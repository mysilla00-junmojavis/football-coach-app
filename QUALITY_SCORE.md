# Quality Score

## 에이전트 코드 품질 기준

### 1. 타입 안전성
- TypeScript 타입 명시율 80% 이상
- any 남용 금지 (불가피한 경우 주석 필수)

### 2. 컴포넌트 크기
- 단일 컴포넌트 200줄 이하
- 위반 시 반드시 분리

### 3. 에러 처리
- 모든 Supabase 쿼리는 try/catch 또는
  error 핸들링 필수
- 사용자에게 에러 상태를 반드시 노출
  (toast 또는 에러 메시지)

### 4. 로딩 상태
- 데이터 페칭이 있는 모든 UI는
  로딩 상태(Skeleton) 필수

### 5. 접근성
- 모든 버튼/링크에 aria-label 또는
  텍스트 레이블 필수
- 이미지에 alt 텍스트 필수

## PR 체크리스트 (에이전트용)
- [ ] 타입 에러 없음
- [ ] 로딩/에러 상태 처리됨
- [ ] 모바일(375px)에서 레이아웃 깨지지 않음
- [ ] 콘솔 에러 없음
