# Reliability

## 목표
MVP 단계에서 서비스 안정성 기본 기준을 정의한다.

## Supabase
- RLS(Row Level Security) 반드시 활성화
- 코치는 자신의 데이터만 읽기/쓰기 가능
- 타인의 drills, plans, records 접근 불가

## 에러 복구
- 네트워크 오류 시 사용자에게 명확한 메시지 표시
- 폼 제출 실패 시 입력값 유지
- 페이지 새로고침 시 로그인 상태 유지

## 데이터 보호
- 삭제 액션은 반드시 확인 다이얼로그 표시
- Supabase 무료 티어 제한 모니터링
  - DB: 500MB 이하
  - 파일 스토리지: 1GB 이하

## MVP 제외 항목 (v2.0 후보)
- 자동 백업
- 99.9% 업타임 SLA
- 에러 모니터링 (Sentry 등)
